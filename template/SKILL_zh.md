# 技能名称

对该技能涵盖的内容以及何时使用提供简明扼要的描述。该描述会出现在技能发现中，并应有助于助手（和人类）确定此技能何时相关。

**重要提示**：为了获得最佳性能，请将 SKILL.md 的正文控制在 500 行以内。将详细的参考资料移动到 `references/` 目录下的单独文件中。

## 何时激活

描述应激活此技能的具体情况、任务或上下文。包括直接触发词（特定关键词或任务类型）和间接信号（表明该技能相关的更广泛模式）。

请使用第三人称编写。描述会被注入到系统提示词中，不一致的视角可能会导致技能发现问题。

- 正确示例："处理 Excel 文件并生成报告"
- 避免示例："我可以帮你处理 Excel 文件"

## 核心概念

解释该技能涵盖的基本概览。这些是该技能传授的心理模型、原则或框架。

默认假设：Claude 已经非常聪明。只需添加 Claude 尚不具备的背景信息。对每条信息进行审视：
- "Claude 真的需要这个解释吗？"
- "我可以假设 Claude 已经知道这一点吗？"
- "这段文字值得消耗这些 Token 吗？"

## 详细主题

### 主题 1

对第一个主要主题提供详细解释。包括具体技术、模式或方法。使用示例来说明概念。

### 主题 2

对第二个主要主题提供详细解释。根据需要增加更多主题。

对于较长的主题，考虑将内容移至 `references/` 并建立链接：
- 参见 [详细参考](./references/topic-details.md) 以获取完整实现

## 实践指南

提供应用该技能的可操作指南。包括常用模式、应避免的反模式，以及在不同方法之间进行选择的决策框架。

根据任务的脆弱程度匹配精确度：
- **高度自由**：多种方法均有效，决策取决于具体上下文
- **中度自由**：存在首选模式，一定程度的差异是可以接受的
- **低度自由**：操作是非常脆弱的，必须遵循特定的顺序

## 示例

提供说明技能应用的具体示例。示例应展示前后对比、演示正确用法，或展示如何处理边缘情况。

使用输入/输出对以保持清晰：

**示例：**
```
输入：[描述输入内容]
输出：[展示预期输出]
```
