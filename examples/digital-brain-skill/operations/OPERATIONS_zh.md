# 运营模块 (Operations Module)

您的个人生产力操作系统。

## 模块内的文件

| 文件 | 格式 | 用途 |
|------|--------|---------|
| `todos.md` | Markdown | 活跃任务列表 |
| `goals.yaml` | YAML | OKR 与目标跟踪 |
| `meetings.jsonl` | JSONL | 会议记录与笔记 |
| `metrics.jsonl` | JSONL | 关键指标跟踪 |
| `reviews/` | 文件夹 | 周复盘/月复盘 |

## 工作流

### 每日流程
```
1. 早晨：查看 todos.md，确定优先级。
2. 全天：勾选已完成项，添加新任务。
3. 晚上：记录会议，更新指标。
```

### 每周复盘 (每周日运行)
1. 运行 `agents/scripts/weekly_review.py`。
2. 复核已完成任务与计划任务。
3. 检查 metrics.jsonl 中的指标。
4. 规划下一周的优先级。
5. 在 goals.yaml 中更新进度。

### 设定目标 (每季度)
1. 复盘上一季度目标。
2. 在 goals.yaml 中更新新的 OKR。
3. 将其分解为每月目标。
4. 使内容日历与目标保持一致。

## 智能体指令

<instructions>
管理运营事务时：

1. **待办事项**：使用优先级 (P0-P3)，保持列表最新。
2. **目标**：在重大决策或规划前予以参考。
3. **会议**：结束后立即记录关键收获。
4. **指标**：至少每周更新一次。
5. **复盘**：应产出见解，而不仅仅是摘要。

优先级分级：
- P0: 今天必做，会阻塞其他工作。
- P1: 本周必做，重要项。
- P2: 本月必做，有价值项。
- P3: 暂存，锦上添花项。

当被要求协助规划或排列优先级时：
1. 检查 goals.yaml 以确保目标一致性。
2. 审核 todos.md 中的现有工作量。
3. 考虑时间紧迫性和依赖关系。
4. 建议合理的截止时间。
</instructions>

## 生产力原则

```yaml
原则:
  - "无情地排列优先级，拒绝瞎忙"
  - "对于 P1-P3 任务，完成 > 完美"
  - "将类似任务分配在同一批次处理"
  - "保护深度工作时间"
  - "每周复盘是不可商榷的"
```
