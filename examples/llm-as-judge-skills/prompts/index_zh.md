# 提示词索引 (Prompts Index)

提示词 (Prompts) 是可复用的模板，定义了智能体和工具与 LLM 交互的方式。

## 提示词类别

### 评估提示词 (Evaluation Prompts)
**路径**：`prompts/evaluation/`

用于质量评估任务的模板。

| 提示词 | 用途 | 使用者 |
|--------|---------|---------|
| `direct-scoring-prompt` | 评估单个响应 | 评估者智能体, directScore 工具 |
| `pairwise-comparison-prompt` | 对两个响应进行对比 | 评估者智能体, pairwiseCompare 工具 |

---

### 研究提示词 (Research Prompts)
**路径**：`prompts/research/`

用于信息收集和综合的模板。

| 提示词 | 用途 | 使用者 |
|--------|---------|---------|
| `research-synthesis-prompt` | 综合研究发现 | 研究员智能体 |

---

### 智能体系统提示词 (Agent System Prompts)
**路径**：`prompts/agent-system/`

用于智能体定义的系统提示词。

| 提示词 | 用途 | 使用者 |
|--------|---------|---------|
| `orchestrator-prompt` | 多智能体协调 | 编排者智能体 |

## 提示词模板格式

### 标准结构

```markdown
# 提示词名称

## 目的
简要描述该提示词实现的目标。

## 提示词模板
```markdown
[包含 {{变量}} 的实际提示词内容]
```

## 变量
| 变量 | 描述 | 是否必需 |
|----------|-------------|----------|
| 变量名 | 包含的内容 | 是/否 |

## 使用示例
展示输入和预期输出的具体示例。

## 最佳实践
有效使用此提示词的指南。
```

### 变量语法

使用 Handlebars 风格的模板语法：

```markdown
{{变量}}                 # 简单替换
{{#if 条件}}...{{/if}} # 条件分支
{{#each 数组}}...{{/each}} # 循环迭代
```

## 提示词设计原则

### 1. 清晰的角色定义
明确告知模型它是谁以及它正在做什么。

```markdown
你是一位专业评估专家，负责评估 AI 生成响应的质量。
```

### 2. 明确的指令
不要假设模型会自动推断出要求。

```markdown
针对每一项标准：
1. 首先，从响应中识别出具体证据。
2. 随后，根据评分量表确定合适的分数。
3. 最后，提供具备可操作性的反馈。
```

### 3. 结构化输出
指定你所需的精确格式。

```markdown
请将你的响应格式化为结构化的 JSON：
```json
{
  "scores": [...],
  "summary": {...}
}
```
```

### 4. 护栏 (Guard Rails)
包含限制条件和警告信息。

```markdown
重要指南：
- 不要仅仅因为响应较长就偏好它。
- 不要根据响应的位置（A 还是 B）来产生偏好。
- 专注于指定的评估标准。
```

## 添加新提示词

1. 确定类别或创建新类别：`prompts/<类别>/`
2. 创建提示词文件：`prompts/<类别>/<提示词名称>.md`
3. 包含以下内容：
   - 目的
   - 包含变量的模板
   - 变量文档
   - 使用示例
   - 最佳实践
4. 更新此索引文件。

## 提示词测试检查清单

- [ ] 变量是否能正确渲染。
- [ ] 输出格式是否可解析。
- [ ] 是否处理了边缘情况。
- [ ] 指令是否存在歧义。
- [ ] 示例是否匹配预期输出。
- [ ] 限制条件是否清晰明确。
