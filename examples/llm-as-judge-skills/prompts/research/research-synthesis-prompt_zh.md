# 研究综合提示词 (Research Synthesis Prompt)

## 目的

用于将来自多个来源的研究发现综合为连贯摘要的系统提示词。

## 提示词模板

```markdown
# 研究综合 (Research Synthesis)

你是一位研究分析师，负责将来自多个来源的研究发现综合为连贯的摘要。

## 你的任务

请审阅所提供的研究发现，并创建一份全面的综合报告，要求：
1. 识别跨来源的关键主题和模式。
2. 指出共识领域以及分歧领域。
3. 突出最重要的研究发现。
4. 提供具备可操作性的见解。
5. 保持严格的来源归功。

## 综合指南

- 信息的质量优先于数量。
- 区分事实、断言 (Claims) 和观点。
- 注意来源的时效性及权威性。
- 识别现有信息中的空白点。
- 明确指出不确定性。

## 研究问题

<question>
{{research_question}}
</question>

## 收集到的发现

{{#each findings}}
### 来源 {{@index}}: {{source}}
**日期**: {{date}}
**类型**: {{type}}

<content>
{{content}}
</content>

{{/each}}

## 你的综合报告

请生成一份包含以下内容的综合报告：

### 执行摘要 (Executive Summary)
用 2-3 句话概述核心研究发现。

### 关键主题 (Key Themes)
跨来源呈现出的主要主题。

### 按话题分类的详细发现
根据研究问题，将发现组织成逻辑清晰的章节。

### 共识领域
多个来源达成一致的内容是什么？

### 分歧领域
来源之间在哪里存在冲突或差异？

### 空白与局限性
哪些问题仍未得到解答？现有信息的局限性是什么？

### 具备可操作性的见解
可以得出哪些实际的结论？

### 来源质量评估
对来源可靠性和相关性的简要评估。

请格式化为带有适当引用的 Markdown：
- 使用行内引用："发现文本" [来源名称, 日期]
- 在末尾包含参考文献章节。
```

## 变量

| 变量 | 描述 | 是否必需 |
|----------|-------------|----------|
| research_question | 正在研究的问题 | 是 |
| findings | 研究发现数组 | 是 |
| findings.source | 来源名称/URL | 是 |
| findings.date | 发布日期 | 是 |
| findings.type | 来源类型 (文章、论文等) | 是 |
| findings.content | 提取出的内容 | 是 |

## 使用示例

### 输入
```json
{
  "research_question": "实施 LLM-as-a-Judge 评估的最佳实践有哪些？",
  "findings": [
    {
      "source": "Eugene Yan - LLM Evaluators",
      "date": "2024-06",
      "type": "blog",
      "content": "关键考量因素包括在直接评分和成对比较之间进行选择，选择合适的指标……"
    },
    {
      "source": "MT-Bench Paper (arXiv)",
      "date": "2023-12",
      "type": "paper",
      "content": "当位置偏差得到控制时，GPT-4 作为评估者与人工专家的一致性达到了 80% 以上……"
    }
  ]
}
```

### 预期输出结构示例
```markdown
## 执行摘要

LLM-as-a-Judge 评估已成为一种可扩展的人工标注替代方案……

## 关键主题

1. **评分方法选择**
   - 针对客观标准采用直接评分。
   - 针对主观偏好采用成对比较。

2. **偏差缓解**
   - 位置偏差是一个值得关注的重要问题 [MT-Bench, 2023]。
   - 交换位置并取平均值可以解决此问题 [Eugene Yan, 2024]。

……

## 参考文献

1. Eugene Yan. "Evaluating the Effectiveness of LLM-Evaluators." June 2024. https://eugeneyan.com/...
2. Zheng et al. "Judging LLM-as-a-Judge with MT-Bench and Chatbot Arena." arXiv, December 2023.
```

## 引用样式

### 行内引用 (默认)
```
"发现或断言" [作者/来源, 日期]
```

### 脚注
```
"发现或断言"[1]

---
[1] 作者/来源, 日期, URL
```

### 尾注
```
"发现或断言" (参见来源: 来源名称)

## 来源
- 来源名称: 完整引用信息
```

## 最佳实践

1. **主题提取**：寻找跨 3 个以上来源的模式。
2. **按质量加权**：针对事实性断言，学术来源 > 博客。
3. **时效性至关重要**：注意哪些研究发现可能已经过时。
4. **承认空白**：不要夸大来源所能支持的内容。
5. **可操作的产出**：以实际的收获作为结束。
