# 技能索引 (Skills Index)

技能 (Skills) 是基础性的知识模块，为智能体、工具和提示词的设计与实施提供指导。

## 可用技能

### LLM 评估者 (LLM Evaluator)
**路径**：`skills/llm-evaluator/llm-evaluator.md`

涵盖了 LLM-as-a-Judge 评估方法论，包括：
- 评分方法（直接评分、成对比较、基于参考的评分）。
- 评估指标（分类准确率、相关性等）。
- 已知的偏差及缓解策略。
- 实施模式。

**核心要点**：
- 针对客观评估使用直接评分。
- 针对主观偏好使用成对比较。
- 务必缓解位置偏差。
- 优先选择分类指标以提高可解释性。

### 上下文基础 (Context Fundamentals)
**路径**：`skills/context-fundamentals/context-fundamentals.md`

涵盖了上下文工程原理，包括：
- 上下文窗口管理。
- 信息分层。
- 上下文类型（静态、动态、瞬态）。
- 相关性过滤。

**核心要点**：
- 按优先级组织上下文结构。
- 明确表达，而非含混隐晦。
- 去除冗余内容。
- 标注信息的时效性。

### 工具设计 (Tool Design)
**路径**：`skills/tool-design/tool-design.md`

涵盖了智能体工具设计的最佳实践，包括：
- 单一职责原则。
- 输入/输出 Schema。
- 错误处理模式。
- AI SDK 6 新特性（批准、严格模式、示例）。

**核心要点**：
- 清晰且经过验证的 Schema。
- 可预测的输出结构。
- 优雅的错误处理。
- 对危险工具考虑引入批准机制。

## 添加新技能

1. 创建技能目录：`skills/<技能名称>/`
2. 创建主文件：`skills/<技能名称>/<技能名称>.md`
3. 包含以下内容：
   - 概述与目的
   - 核心原则
   - 实践模式
   - 实施示例
   - 参考文献
4. 更新此索引文件。

## 技能开发指南

- 专注于可跨实现方案迁移的通用原则。
- 包含具体的示例和模式。
- 引用权威的信息来源。
- 保持内容具备可操作性，而非仅仅是理论。
- 随着理解的深入不断更新内容。
